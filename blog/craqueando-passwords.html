Quebrando o password da BIOS

Existem programas que são capazes de descobrir o password da Bios, eles agem diretamente com o setup do PC, abaixo estão alguns programas próprios para "quebrar" password da Bios:

  • amipsw.com
  • ami.com
  • aw.com
  • amidecod.com

Quebrando o password do Windows

Windows 3.xx

Para "quebrar" o password do Windows 3.xx você deve apertar CTRL+ALT+DEL depois tecle ENTER na caixa de verificação, depois que você passou do password, apague as linha run=??? e load=??? do arquivo Win.ini que está dentro do diretório do Windows (pode usar até mesmo o Notepad ou o Edit do MS-DOS).

Windows 95

Para "quebrar" o password no Windows 95 é quase do mesmo modo do Windows 3.xx, quando aparecer a caixa de verificação do password aparecer aperte CTRL+ALT+DEL, escolha a janela do Password na lista depois clique em "Finalizar tarefa", depois que passou do password, apague as linha run=??? e load=??? do arquivo Win.ini que está dentro do diretório do Windows (pode usar até mesmo o Notepad ou o Edit do MS-DOS).

Retirando o password da Proteção de Tela

Windows 3.xx

Apague tudo que tiver na área de password do arquivo control.ini que está dentro do diretório do Windows (pode usar até mesmo o Edit do MS-DOS).

Windows 95

Renomeie todos os arquivos com extensão .pwl por um nome qualquer e anote em um papel as alterações feitas para caso ocorra algum problema.

Quebrando password dos documentos do Word

Para "quebrar" o password de um documento do Word existem programas feitos por cracker específicos para isso, veja abaixo:

wfwcd.zip

Quebrando password de arquivos ZIP

Para "quebrar" o password de um arquivo ZIP, você deve usar algum desse programas abaixo:

zipcrack.zip
fzc104.zip

Quebrando pasword de arquivos ARJ

Para "quebrar" o password de um arquivo ARJ, você deve usar algum desses programa abaixo:

brkarj10.zip
arjb1utg.zip

                        

Acima foram mencionados métodos que utilizam programas e algumas estratégias específicas. Agora, descreverei um método bastante eficaz:

Provavelmente, você já percebeu que na sua escola existe algum tipo de rede conectando o computador central aos computadores dos alunos (ao menos, é assim aqui). No entanto, essa rede geralmente só está acessível aos professores com mais experiência, devido à existência de senhas para acesso. É provável que você já tenha imaginado como devem existir documentos importantes, provas, notas dos alunos, entre outros dados nesse computador central. E, talvez, tenha se perguntado: "Como seria possível acessar esse conteúdo?" Muitos já pensaram nisso em algum momento.

Existe um método para descobrir essas senhas, mas ele exige paciência e estratégia. Não será algo que você conseguirá de imediato; será necessário persistência e cuidado. Descreverei o processo de forma geral, mas ressalto: pense nas consequências antes de tentar algo assim.

Primeiramente, você precisará obter um dos programas listados abaixo:

  • DOS: keycopy.zip
  • Windows 95: keytrap.zip

Esses programas registram, em um arquivo, todas as teclas digitadas desde o momento em que são executados. A partir disso, acredito que você já entendeu o propósito, certo?

Depois de obter o programa desejado, transfira-o para um disquete em sua forma descompactada. Leve-o para a escola no dia em que tiver uma aula na sala de informática. Durante um momento em que o professor esteja ocupado atendendo outros alunos, insira o disquete no drive do computador e copie os arquivos para o diretório do Windows. Isso pode ser feito pelo Prompt do DOS com o seguinte comando:

copy a:\*.* c:\<diretório_do_windows>

Certifique-se de copiar o conteúdo do arquivo correspondente ao programa (KeyCopy.zip) para que ele seja executado automaticamente quando o sistema for iniciado, sem levantar suspeitas. Após realizar a cópia para o diretório do Windows, acesse a raiz do sistema e edite o arquivo autoexec.bat (utilizando o editor de texto do MS-DOS) para incluir as seguintes linhas:

cd\windows
<nome_do_programa_do_disquete> # Geralmente é KeyCopy

Salve as alterações, saia do prompt e continue agindo normalmente, como se estivesse concentrado na aula. Este procedimento deve ser realizado rapidamente, idealmente em menos de um minuto, para evitar que seja notado.

Na próxima vez que usar o mesmo computador, volte ao diretório do Windows pelo DOS e copie o arquivo gerado pelo programa (aquele que contém o registro das teclas digitadas) para o disquete. Em casa, você poderá analisar o arquivo com calma, procurando por informações relacionadas à senha.

Se você chegou a este ponto, o restante do processo será mais simples. Porém, é essencial ser discreto ao acessar informações do computador central e lembrar-se dos possíveis riscos e consequências.
