  • Honre as boas práticas de código

    Escrever um código limpo, legível e bem comentado é uma obrigação, não uma escolha.

  • Crie um README

    Um projeto sem documentação é um projeto morto.

  • Não dê commit sem uma mensagem significativa

    `git commit -m "fix"` ou `final_version_final(2).zip` são exemplos de mensagens ruins.

  • Não code sem testar

    A testabilidade é a alma do desenvolvimento bem-feito.

  • Não crie um `if` sem `else`

    Nem todo cenário é feliz. Preveja os caminhos obscuros.

  • Não copie código alheio simplesmente, mas aprenda com ele

    Copiar sem entender é o mesmo que colar sem saber o que está colando.

  • Não multiplique variáveis sem propósito

    Cada nome no código deve ter um motivo para existir.

  • Não esqueça de fechar conexões e liberar recursos

    Recursos abertos sem necessidade são como portas abertas para invasões.

  • Não idolatre frameworks ou linguagens

    Tudo tem sua utilidade e seu momento. Não há bala de prata.
